# P0-00 架构边界冻结

来源：总规划书第 14、16、19、20 节。

## 本文档定位

本文档只负责冻结 P0 阶段的架构边界和任务基线，不负责实现任何功能，不展开为总重构计划。

从 P0-01 到 P0-07 的所有任务文档，都必须把本文档作为共同前置约束；如拆分文档与总规划书冲突，以总规划书为准；如拆分文档之间冲突，以本文档的 P0 边界解释为准。

## 目标

把当前阶段 Do / Don’t Do 固化为开发基线，防止后续任务把 LTClaw 扩成中心化协同平台或服务端审批系统。

## Do / Don't Do 基线

### 当前阶段必须明确做的事

- [ ] 只冻结本地智能策划工作台 + 管理员知识底座的职责边界。
- [ ] 只确认普通策划可在受限能力下写回真实源表。
- [ ] 只确认 Formal Map、Current Release、正式 RAG 仍由管理员手动维护和发布。
- [ ] 只确认正式知识查询入口收口为 Map-gated RAG。
- [ ] 只确认所有 LLM 调用统一走模型配置 / Model Router。
- [ ] 只为后续 P0 任务提供统一边界基线、引用要求和验收口径。

### 当前阶段明确不做的事

- [ ] 不把 LTClaw 扩成中心化协同平台。
- [ ] 不把 P0-00 扩写成 P1/P2/P3 的总重构实施方案。
- [ ] 不实现 SVN 隐藏、RAG、Model Router、Workbench 写回、知识发布等具体功能。
- [ ] 不重新纳入服务端账号、审批流、文件锁、自动备份。
- [ ] 不允许普通策划更新 Formal Map、Current Release 或正式 RAG。
- [ ] 不允许普通策划通过 Draft / Proposal 反向污染正式知识底座。

## Checklist

- [ ] 确认 LTClaw 定位为本地智能策划工作台，不是中心化协同平台。
- [ ] 确认 SVN 只作为团队外部流程，LTClaw 当前版本不执行 SVN 操作。
- [ ] 确认普通策划可以写回真实源表，但不能更新 Formal Map、Current Release 或正式 RAG。
- [ ] 确认管理员手动更新 Map/RAG/Release。
- [ ] 确认当前阶段不做新增字段、新增表。
- [ ] 确认禁止删表、改表名、改表路径、改主键、改 header/comment row。
- [ ] 确认不做服务端账号系统。
- [ ] 确认不做服务端审批流。
- [ ] 确认不做文件锁。
- [ ] 确认不做自动备份。
- [ ] 确认不做普通用户自动知识过期检测。
- [ ] 确认数据整理脚本生成不进入当前阶段，后续放入 MCP 工具池。
- [ ] 确认 KB 不再作为正式知识系统。
- [ ] 确认正式知识查询只保留 Map-gated RAG。
- [ ] 确认所有 LLM 调用统一走模型配置 / Model Router。
- [ ] 确认 Map / RAG / Chat / Workbench 不允许单独配置 API。

## 后续任务引用规则

- [ ] P0-01 至 P0-07 每份文档都要显式引用“遵循 P0-00 架构边界冻结”。
- [ ] 涉及真实源表写回的任务，必须写明仅可引用已有 capability 名称 `workbench.source.write`，且不包含知识底座更新。
- [ ] 涉及知识发布或正式知识边界的任务，必须写明仅可引用已有 capability 名称 `knowledge.build` / `knowledge.publish` / `knowledge.read`，且普通策划不获得发布权限。
- [ ] 涉及模型调用的任务，必须写明不新增 capability，只统一调用路径到 Model Router。
- [ ] 涉及 legacy 模块的任务，必须写明“仅兼容 / 降级 / 迁移，不回升为正式主链路”。

## 输出物

- [ ] 当前阶段 Do / Don’t Do 清单定稿。
- [ ] 后续功能拆分基线定稿。
- [ ] 每个后续任务都引用本边界文档。

## 禁止范围

- [ ] 不引入服务端账号系统。
- [ ] 不引入审批流、文件锁、自动备份。
- [ ] 不新增普通工作台 schema ops。
- [ ] 不让 Draft / Proposal 进入 Formal Map、Release 或正式 RAG。

## 验收标准

- [ ] 新增任务文档均明确引用 P0 边界。
- [ ] 任何涉及写源表、知识发布、模型调用的任务都有对应 capability 和不做范围。
- [ ] 任何涉及写源表、知识发布、模型调用的任务都写清 legacy 兼容或迁移定位。
- [ ] 没有任务要求普通策划更新知识底座。
- [ ] 没有任务暗示普通策划可以更新 Formal Map、Current Release 或正式 RAG。
- [ ] 没有任务把 SVN、服务端账号、审批流、文件锁、自动备份重新纳入当前阶段。

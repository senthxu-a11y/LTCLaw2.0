### P0-1 注入 ai_summary 到 suggest 上下文

## 4. 关键 API 字段变化

### `POST /api/agents/{agentId}/game/workbench/suggest`

请求新增字段：
```jsonc
{
  "user_intent": "string",
  "context_tables": ["Equipment"],
  "current_pending": [],
  "chat_history": [
    { "role": "user", "content": "上一轮问的内容" },
    { "role": "assistant", "content": "上一轮回复" }
  ]
}
```

响应新增字段：
```jsonc
{
  "message": "...",
  "changes": [...],
  "context_summary": {
    "main_tables": ["Equipment"],
    "related_tables": [],
    "matched_columns": { "Equipment": ["Quality"] },
    "query_terms": ["品质", "黑铁头盔"]
  }
}
```

## 5. 已知缺陷

1. 依赖图未建表时 `related_tables` 永远空（Equipment 当前就是这种情况）。需要先跑一次 `dependency_resolver`。
2. `query_terms` 的中文 2-gram 噪声较多（如"盔品""质改"），暂未做停用词过滤；当前不影响主要召回。
3. 大表 5000 行截断后再按 `extra_columns` 召回，理论上极端情况会丢失 1500 行外的命中行——后续按 query_terms 优先过滤。
